Linked Dataの話

  • Linked Dataは何？
  • 世の中にある物事を全てURIを与えること。
  • 呂康豪　は　<http://dig.csail.mit.edu/People/kennyluck#I>
  • ティム・バーナーズ＝リー　は　<http://www.w3.org/People/Berners-Lee/card#i>
  • W3C　は　<http://www.w3c.org/data#W3C>
  • 私たちの分散情報グルプ 　は　<http://dig.csail.mit.edu/data#DIG>
  • Tabulator　は　<http://dig.csail.mit.edu/2005/ajar/ajaw/data#Tabulator>
  • URIは何？魔法の識別子。（他の参考：萩野先生の授業講義）
  • 皆さんはもうURI（WebID）を持っていらっしゃいますか？
  • 皆さんの組織は？プロジェクトは？

ホームページがあるけど

  • 情報はコンピューターさえ読めるように書いてあるんですか？（RDFaでできる）
  • RDFをもっと作りましょう！
  • 矛盾のないウェブを建てたいですから、ページとページの内容の物事を区別した方がいいです。（例えば、<http://dig.csail.mit.edu/People/kennyluck>は二歳ぐらいで、<http://dig.csail.mit.edu/People/kennyluck#I>はもう二十一歳）

Linked Dataの現在状況

  • おすすめのスライド、ティムが作った。（他に、武田さんの技術的な日本語の説明スライド）
  • 二十年前にウェブが始まった時の状況に似てると思う。でも、今回はHTMLじゃなく、RDFだ。
  • これこそ本当のWeb 2.0だって、ティムが言ってた。
  • 米国政府とイギリス政府は政府資料をLinked Dataにすることに決めたそうだ。
  • もう止まらない。
  • 日本からの資料やデータベースはもし公開できたら、Linked Dataにして、早く他のLinked Dataに連結しましょう。

私たちの解決方法（その一）

  • RDFを使う
  • は
  • HTML画面はRDF資料の通り作られた。

私たちの解決方法（その二）

  • コメントしたいなら
  • 新しいRDF三つ組（triple）をつける。
  • HTML画面は青い三つ組だけで作られたから、新しい三つ組をつけてもHTML画面に影響しない。
  • でも、使用者は新しく書いた情報を手に入れる可能性もある（RDFで）。

プロトコルについて

  • SPARQLの拡張のSPARQL Updateの使う
  • PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
INSERT { <http://example.org/Dennis#me> rdfs:comment "最低"@ja . }
  • RDFの図の変化しか送らない。
  • PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
DELETE { <http://example.org/Dennis#me> rdfs:comment "最低"@ja . }
  • SELECTやWHERE と組み合わせることもできる。

Tabulatorのことについて

  • 一番重要な述語はrdfs:label
  • URIは、コンピューターとしての概念の名前 。rdfs:label は人間としての概念の名前。
  • Tabulatorの信条の一つ、「人間は人間の名前を使うが良い」

  • 自分は好きな言語を使っていいです。自動翻訳があったら、、
  • RDFは「簡単な人間言語」と呼べます。
  • （RDFは「アのイはウ」の構造だけありますから。名詞はURI、どの国かの言語じゃない）
